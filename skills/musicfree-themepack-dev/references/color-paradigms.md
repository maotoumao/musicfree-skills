# 色彩推导范式

本文档提供多种色彩推导策略的方向性指导。AI 根据用户意图选择合适的范式，具体数值由 AI 自行判断。

## 通用原则

1. **对比度优先**：文本色与背景色之间必须有足够的视觉区分度，确保可读性
2. **品牌色一致性**：`--color-fill-brand`、`--color-text-brand`、`--color-border-brand` 应基于同一色相
3. **on-brand 可读性**：`--color-text-on-brand` 必须在 `--color-fill-brand` 背景上清晰可读
4. **透明度递进**：同一色系的不同强度通过透明度递进实现（如 neutral: 0.1 → hover: 0.16 → active: 0.2）
5. **海拔递进**：Background 类变量从 base（最底层）到 popover（最顶层）逐层递进，体现视觉层级

## 暗色 vs 浅色的对偶关系

MusicFree 默认主题为暗色。暗色与浅色主题在色彩构建上存在系统性对偶：

| 特征 | 暗色主题 | 浅色主题 |
|------|---------|---------|
| 基底色 | 深色（L 5-15%） | 浅色（L 92-98%） |
| 填充/边框基色 | `rgba(255, 255, 255, x)` | `rgba(0, 0, 0, x)` |
| 海拔递进方向 | 越高越亮 | 越高越白或更不透明 |
| 文本主色 | 白色 or 接近白色 | 深灰 or 接近黑色 |
| 阴影强度 | 较重（alpha 0.25-0.5） | 较轻（alpha 0.04-0.15） |
| overlay | 深色半透明 `rgba(0,0,0,0.6)` | 深色半透明（仍用暗色遮罩） |
| on-brand | 深色（与亮品牌色对比） | 白色（与深品牌色对比） |

**关键差异：浅色主题必须覆盖所有颜色类变量。** 因为默认值全部是暗色系的 `rgba(255,255,255,x)`，不覆盖会导致白色填充/边框出现在浅色背景上，几乎不可见。

## 范式 A：单品牌色推导

**适用场景：** 用户给出一个品牌色（如"粉色""#5B6DCD"），需要推导出完整色板。

**推导方向：**

1. **品牌色** → 确定 H(色相)、S(饱和度)、L(亮度)
2. **背景色系列**：
   - 取品牌色的 H，大幅降低 S（5-15%），依据明暗调调整 L
   - 可微量偏移 H 让背景带有品牌色的"色温倾向"
   - 暗色主题 L 从低到高递进；浅色主题 L 从高到低递进
3. **文本色系列**：根据背景亮度选择对立色系，通过透明度分级
4. **填充色系列**：品牌色不同透明度（brand-muted 通常 0.12-0.2）
5. **边框色系列**：主色系的极低透明度
6. **状态色**：通常保持默认值；如果品牌色与某个状态色（红/黄/蓝）色相冲突，需要偏移该状态色

## 范式 B：情绪/氛围词推导

**适用场景：** 用户用词汇描述想要的感觉（如"温暖""赛博朋克""清新自然"）。

**常见情绪到色彩的映射方向：**

| 情绪关键词 | 色温倾向 | 饱和度倾向 | 品牌色方向 | 背景风格 |
|-----------|---------|-----------|-----------|---------|
| 温暖、柔和 | 暖色（橙/琥珀） | 中等 | 暖橙/琥珀 | 深暖灰 |
| 冷冽、专业 | 冷色（蓝/灰蓝） | 低-中 | 冷蓝/冰蓝 | 深冷灰 |
| 清新、自然 | 绿色系 | 中等 | 翠绿/薄荷 | 浅色为主 |
| 赛博朋克 | 冷色 + 高对比霓虹 | 高 | 霓虹紫/霓虹蓝 | 极深暗 |
| 浪漫、梦幻 | 粉紫色系 | 中-高 | 玫瑰粉/薰衣草 | 偏深紫 |
| 简约、高级 | 中性无彩色 | 极低 | 灰色/金色点缀 | 纯灰色系 |
| 活力、年轻 | 暖色 + 高饱和 | 高 | 橙/珊瑚/亮绿 | 渐变背景 |
| 复古、怀旧 | 暖色偏棕 | 中低 | 褐色/暗金 | 纸质感 |

AI 根据用户描述选择最匹配的方向，并在向用户展示方案时解释选择理由。

## 范式 C：参考图/参考应用提取

**适用场景：** 用户提供了参考图片、截图、或提到了某个应用的配色。

**推导方向：**

1. 从参考中识别 3-5 个主要色彩
2. 判断主色（面积最大）、辅助色、点缀色
3. 将主色映射为品牌色或背景色（取决于它在参考中的角色）
4. 将辅助色/点缀色用于品牌色或背景渐变
5. 保持参考的整体色温和饱和度风格

## 范式 D：对偶反转

**适用场景：** 基于一个已有的暗色（或浅色）主题，生成对应的浅色（或暗色）版本。

**反转规则：**
- 背景色：H 保持，S 微调，L 翻转到对偶区间
- 填充/边框基色：`rgba(white, x)` ↔ `rgba(black, x)`（透明度可微调）
- 文本色：从亮色系切换到暗色系（或反向）
- 阴影：暗→浅时降低 alpha；纯黑→none 可保持 none
- 品牌色：可能需要微调亮度以适应新背景
- overlay：通常不变（遮罩始终用暗色）

## 特殊风格处理

### AMOLED 纯黑主题

- `--color-bg-base` 设为 `rgb(0, 0, 0)` 或极深（L ≤ 2%）
- 所有 shadow 设为 `none`（纯黑上阴影无意义）
- background-image 设为 `none`
- 使用高对比度的品牌色（深色背景需要亮颜色）
- surface/raised 使用低透明度白色叠加（让层级可感知但不刺眼）

### 玻璃态（Glassmorphism）

- 背景色使用半透明值（alpha 0.3-0.7）
- 需要配合背景图或 iframe 动态背景才有效果
- 适当增大 blur 值
- 边框使用低透明度白色增强玻璃边缘感

### 扁平化（Flat Design）

- 所有 shadow 设为 `none`
- 所有 blur 设为 `0px`
- 背景色使用不透明的纯色
- 使用实色（而非透明度）区分层级

## 色彩描述指南

向用户描述色彩时，同时提供以下信息：

1. **色值**：如 `rgb(167, 139, 250)` 或 `#a78bfa`
2. **直观描述**：如"薰衣草紫"
3. **类比参考**（可选）：如"类似 Discord 的紫色"

避免只给色值而不解释——用户可能无法从 `#34d399` 直观想象出"翡翠绿"。
